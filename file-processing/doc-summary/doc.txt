C. Notion of Signal Processing:

Each hardware, such as keyboard, mouse, have their own filters. When the data is sent to the device memory through driver, the driver implements another set of signal processing. Then the data reaches to the kernel, where another level of signal processing is performed, and finally data is reached to the application program.

If you have worked with .Net, then user.dll, keyboard.dll, kernel.dll, all can be trapped for keyboard and mouse data, and all data will return different sets of data. On the other hand, in a typical C#.Net application, when you use Keyboard events such as keyUP or KeyDown, you get access to the events, and not raw data.

Therefore, in information technology, every single hardware and software layer sees the input as an analog data, and processes it based on its own needs and requirements.
